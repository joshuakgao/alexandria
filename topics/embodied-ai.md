## Overview

*Last updated: 2026-06-19 | Sources: 27 papers*

## Current thesis

The embodied memory landscape is splitting along four axes: **non-parametric vs. parametric memory**, **navigation/QA vs. manipulation** tasks, **accuracy vs. efficiency**, and now **map-based vs. map-free localization**. The dominant paradigm for navigation and EQA builds non-parametric memory stores (vector databases, scene graphs, semantic forests, 4D object databases) queried via LLM/VLM pipelines—converging on retrieval-augmented approaches that outperform fixed-context-window methods. However, Chameleon challenges this paradigm for manipulation, arguing that semantic compression and similarity-based retrieval discard fine-grained perceptual cues needed to resolve perceptual aliasing. EmbodiedLGR introduces a third dimension by prioritizing **deployment efficiency**: using lightweight VLMs (Florence-2 at <1B params) and hybrid graph+vector DB memory to achieve real-time responsiveness on robot hardware, accepting lower accuracy as a trade-off. This reveals that most systems in the topic implicitly assume unlimited compute (GPT-4o, SAM2, Gemma3), while practical deployment demands architectures that degrade gracefully under resource constraints. HOV-SG represents the strongest case for structured reconstruction, building four-level open-vocabulary scene graphs that enable multi-floor language-grounded navigation impossible with flat representations. Memory Over Maps adds a new dimension by arguing that 3D reconstruction itself is unnecessary — posed RGB-D keyframes with on-demand depth backprojection can match or exceed reconstruction-based localization while reducing preprocessing by 100x+. BridgeEQA expands the scope further by demonstrating that **professional infrastructure inspection** is a compelling problem class for EM-EQA — demanding multi-scale reasoning over 48+ images per scene, alignment with standardized condition rubrics, and evidence citation — far exceeding the complexity of household EQA benchmarks. Its EMVR method converts EM-EQA to A-EQA via image-based scene graph traversal, demonstrating that navigational context retrieval mitigates the "lost in the middle" problem in long-context VLMs. The optimal memory architecture depends on five factors: task type (navigation vs. manipulation vs. inspection), required accuracy, available compute budget, whether persistent 3D state is actually needed, and the scale/complexity of the scene being reasoned about.

## Key trends

- **Dynamic environment QA as evaluation paradigm**: Env-QA (Gao et al., ICCV 2021) established the first video QA benchmark purely focused on dynamic environment understanding — tracking object state changes across interaction events in AI2-THOR. The event-level video representation (content-based temporal segmentation rather than fixed-interval sampling) and the five question types (attribute, state, event, order, number) provided the evaluation framework that later embodied memory systems (Embodied VideoAgent, R⁴, Mind Palace) build upon. The 47% accuracy ceiling revealed that long-term state tracking and multi-event temporal reasoning are the primary bottlenecks — directly motivating the persistent memory architectures that followed.
- **Metric-semantic SLAM foundations**: Kimera (2020, MIT SPARK) established the modular perception stack — VIO + robust PGO + mesh + semantic TSDF — that all subsequent scene graph systems build upon. The key design principle of parallelized multi-rate processing (IMU-rate VIO → keyframe meshing → slow-rate semantics) directly shaped Hydra's architecture.
- **Topological graph foundations**: Oleynikova et al. (2018) established the ESDF → GVD → sparse skeleton pipeline that underpins the navigational layer of Hydra, HOV-SG, and MoMa-LLM. The key insight — that the Voronoi diagram of free space provides a compact, topology-preserving graph that maximizes obstacle clearance — remains the standard approach for navigational memory in scene graph systems.
- **Action-conditioned scene graphs for interactive exploration**: RoboEXP (Jiang et al., CoRL 2024) extends scene graphs beyond static spatial relations to model action effects — what happens when you open a drawer, what preconditions must be met, and what hidden objects are revealed. The ACSG representation enables robots to reason about partially observable environments where physical interaction is required to discover objects. GPT-4V serves dual roles (action proposer + verifier), while explicit graph memory prevents the LMM from losing track of complex multi-step dependencies. Achieves 70-90% success on real-world tabletop scenarios where a strong GPT-4V baseline achieves 0-30%, even with manual action execution.
- **From metric maps to semantic abstractions**: Traditional dense SLAM and point-cloud maps don't scale and lack semantic queryability. The field is moving toward topological and hierarchical representations that trade geometric precision for semantic richness. HOV-SG exemplifies this with its four-level hierarchy (building→floors→rooms→objects) enriched with CLIP features, achieving 75% storage reduction over dense open-vocabulary maps while enabling multi-level language grounding.
- **RAG adaptation for embodiment**: Text-based RAG techniques (GraphRAG, LightRAG) are being extended to handle the unique properties of embodied data—spatial correlation, multimodal inputs, and observation redundancy.
- **Platform-agnostic memory**: Memory architectures are designed to work across different robotic platforms (drones, quadrupeds, wheeled robots) by abstracting away control-specific details.
- **Surprise-gated episodic memory solves the "when to store" problem**: Worth Remembering (Gorlo et al., MIT SPARK + Lockheed Martin, 2026) introduces Bayesian surprise in V-JEPA-2 latent space as a principled, deployment-agnostic gating mechanism for robotic episodic memory. Rather than fixed-interval storage (which wastes budget on redundant data) or indiscriminate storage (which accumulates noise), surprise-gated memory stores only 1.7% of frames — retaining moments where observations maximally shift the predictive model's expectations. Augmenting DAAAM's 4D scene graph with these sparse episodes yields +12% QA accuracy on OC-NaVQA and SOTA on Generic Event Boundary Detection, outperforming supervised and offline methods with an unsupervised causal approach. This connects world model representations (V-JEPA-2) to memory formation, and draws directly from cognitive science (CLS theory, predictive coding, event segmentation theory).
- **Generative spatial reasoning benchmarking for embodied VLMs**: ESPIRE (Zhao et al., 2026) shifts evaluation from discriminative VQA to a fully generative paradigm where VLMs must produce 3D positions and orientations for robotic pick-and-place tasks in Isaac Sim. The systematic design across 148 spatial reasoning types (4 spatial aspects × 3 reference frames × 2 reference object types × multiple granularities) enables fine-grained diagnosis. Key findings: (1) VLMs perform much better on localization than execution, indicating passive spatial understanding outpaces acting ability; (2) orientation reasoning is the weakest spatial aspect across all models; (3) distance reasoning also poses significant challenges. These findings directly inform what spatial capabilities embodied memory systems need to provide — accurate metric distances and rotational geometry.
- **Inference-time memory editing replaces exhaustive preprocessing**: GraphPad (Ali et al., 2025) demonstrates that mutable scene graphs — where a VLM can add missing objects, relations, and semantic annotations during reasoning via language-callable APIs — outperform static scene graphs while using 5x fewer input frames. The key insight is that pre-built representations inevitably lack task-relevant information; rather than overprovisioning (retaining everything) or rebuilding (expensive), targeted reasoning-guided perception patches specific gaps. The navigation log provides the largest single gain (+9.6pp) by enabling the VLM to identify which frames to re-examine, while the scratch-pad anchors task-specific notes to graph nodes. Average of only 1.9 API calls per question demonstrates high efficiency of targeted perception over brute-force processing.
- **LLM-in-the-loop retrieval**: Moving beyond pure embedding similarity to structured, reasoning-based retrieval using LLMs at multiple levels of abstraction. Chen et al. (2023, MIT SPARK) established the foundation by showing LMs can provide common-sense knowledge for room classification in scene graphs, with informativeness-based object selection as a principled mechanism for choosing what to present to the LM. MoMa-LLM extends this to interactive planning, demonstrating that structured textual encoding of scene graphs (not raw JSON or object lists) is critical for effective LLM reasoning.
- **Spatio-temporal reasoning**: Embodied memory must support not just "what" and "where" queries but also "when" and "how long"—temporal dimensions that static scene representations cannot capture (ReMEmbR).
- **Vector databases as embodied memory backends**: Using scalable vector databases to store multimodal embeddings (text, position, time), enabling efficient approximate nearest-neighbor retrieval over unbounded histories.
- **Object-centric 4D representations**: Moving from scene-level or region-level memory to per-object structured records with semantic, spatial, and temporal fields (R⁴, Embodied VideoAgent), enabling precise metric reasoning and multi-agent sharing. Embodied VideoAgent adds explicit state tracking and action-driven VLM-based updates to per-object entries, addressing the gap between passive recording and dynamic state maintenance.
- **Video understanding agents as embodied memory builders**: Embodied VideoAgent demonstrates that long-form video understanding pipelines (VideoAgent) can be adapted for embodied scene comprehension by fusing egocentric video with depth/pose sensing, constructing persistent object memory rather than ephemeral temporal summaries.
- **Training-free frameworks**: R⁴ demonstrates that structured memory + retrieval can dramatically boost small VLMs without any task-specific training, suggesting that memory architecture may matter more than model scale for embodied reasoning.
- **Active exploration integrated with memory**: Mind Palace shows that embodied memory is not just about recall—the agent must actively decide when to retrieve past observations vs. physically explore the current environment, framing EQA as a planning problem over both memory and physical space.
- **Episodic memory with temporal boundaries**: Rather than merging all observations into a single memory (as in R⁴ and ReMEmbR), Mind Palace preserves episode boundaries, enabling temporal reasoning about how environments evolve across deployments spanning days to months.
- **Images as memory primitives**: SnapMem challenges the assumption that memory must be structured/symbolic, showing that well-chosen raw images (via co-visibility clustering) can outperform scene graphs for spatial reasoning tasks, since VLMs can directly extract nuanced spatial information from images.
- **Exploration-aware memory**: Both SnapMem (frontier snapshots) and Mind Palace (explore action) integrate exploration planning into the memory system itself, representing not just what is known but what remains to be discovered.
- **Bio-inspired differentiable memory**: Chameleon brings neuroscience-grounded design (EC–HC–PFC) to embodied memory, with fully differentiable episodic/working state coupling and goal-directed recall via imagination (HoloHead). This represents a departure from the engineering-driven, LLM-pipeline approach of all other systems in this topic.
- **Perceptual aliasing as a memory problem**: Chameleon highlights that manipulation tasks create perceptual aliasing (identical observations requiring different actions), which cannot be resolved by semantic compression—the dominant approach in navigation/QA-focused systems. This suggests embodied memory research must bifurcate by task type.
- **Sensor-minimal scene graphs**: Mono-Hydra (2024) demonstrates that Hydra's hierarchical scene graph can be built from monocular camera + IMU alone, replacing depth sensors with self-supervised depth estimation (sub-20cm error). This broadens deployment to platforms where RGB-D is impractical (UAVs, lightweight robots), though with accuracy trade-offs.
- **Efficiency-first design**: EmbodiedLGR prioritizes real-time responsiveness and edge deployment over accuracy, using lightweight VLMs (Florence-2 at <1B) and hybrid graph+vector DB memory. Achieves ~50% latency reduction over ReMEmbR by routing simple queries through a fast graph structure and falling back to vector DB only for complex semantic queries.
- **Local scene graph updates for manipulation**: DovSG demonstrates that surgical, voxel-level scene graph updates — identifying obsolete voxels via depth/color comparison and recomputing only the affected subgraph — enable 27x faster adaptation than full reconstruction (ConceptGraphs). This is the first system to validate dynamic scene graph memory for real-hardware mobile manipulation across 240 long-term task trials, showing that memory update granularity directly impacts long-term task success (35% vs. 5% for static-scene baselines).
- **Memory deduplication**: EmbodiedLGR addresses redundancy (a problem acknowledged but unsolved by most systems) through real-time node updates using spatial/semantic thresholds, preventing the monotonic memory growth that plagues vector-DB-only approaches.
- **Information-theoretic retrieval**: STaR applies the Information Bottleneck principle to produce non-redundant, task-relevant evidence sets from multimodal memory. Unlike similarity-based top-K retrieval (which collapses into redundant results from revisited locations), IB clustering ensures one representative per spatial cluster, yielding diverse evidence and 2–4× lower spatial error than ReMEmbR.
- **Tripartite multimodal memory**: STaR's OmniMem (captions + 3D primitives + keyframes) demonstrates that complementary memory modalities capture different information granularities—mid-level dynamics, persistent structure, and fine-grained visual detail—that single-modality stores miss.
- **RL-trained active memory retrieval**: LMEE/MemoryExplorer demonstrates that reinforcement learning can teach agents to proactively invoke memory retrieval tools during exploration, rather than passively constructing memory for post-hoc querying. The multi-task reward (action + frontier + QA + format) with tool-use incentivization is a concrete mechanism for training this behavior.
- **Unifying exploration process and outcome**: LMEE argues that existing benchmarks' focus on task completion outcomes neglects how agents build and use memory during exploration. By combining multi-goal navigation with memory-based QA, LMEE evaluates the cognitive process alongside decision-making, a prerequisite for lifelong learning agents.
- **Implicit memory via internet-scale behavior cloning**: NitroGen (Magne et al., CVPR 2026) represents a counterpoint to explicit memory architectures: a vision-action model trained on 40,000 hours of gameplay across 1,000+ games learns transferable motor primitives (combat, navigation) that function as implicit procedural memory. Single-frame context suffices — no explicit memory store, no retrieval mechanism — yet the model transfers to unseen games. This challenges the assumption that embodied agents require explicit memory systems, at least for reactive tasks where the current observation provides sufficient context. The boundary between "memory needed" (navigation, QA, manipulation) and "memory unnecessary" (reactive gameplay) tasks remains poorly understood.
- **View graphs as empirical world models from exploration**: ViewSuite (Wang et al., 2026) introduces the view graph — an any-view-to-any-view map assembled from on-policy self-exploration trajectories, where nodes are viewpoints with rendered views and edges are actions. Unlike traditional scene graphs that reconstruct spatial structure, the view graph captures the agent's *experienced* view transitions, including from failed trajectories. Through task reformulation, any path becomes a valid planning demonstration. This represents a new form of embodied memory: not storing what was seen (like R⁴/ReMEmbR) but storing how viewpoints connect, enabling planning through view space rather than physical space. The iterative growth (graph expands as policy improves) mirrors the explore-then-consolidate pattern seen in Mind Palace and LMEE.
- **Cognitive maps as explicit belief probing**: Theory of Space (Zhang et al., ICLR 2026) formalizes the evaluation of how agents construct, maintain, and revise internal spatial beliefs (cognitive maps) through active exploration. By requiring agents to externalize their cognitive map at each step, it directly measures belief quality over time — revealing that global beliefs suffer from instability (spatial knowledge degrades even with correct local observations) and belief inertia (agents fail to overwrite obsolete priors after environmental changes). This connects to the embodied memory debate: explicit probing shows that *having* memory is insufficient — the update and revision mechanisms are critical failure points.

## Open problems

- **Online incremental memory building**: How to efficiently update hierarchical memory structures as new observations arrive during exploration, without full reconstruction. R⁴ addresses this partially with continuous updating and object matching, but scalability to very large databases over days of operation remains untested.
- **Dynamic environments**: R⁴ tracks object appearance/disappearance timestamps, Embodied VideoAgent introduces VLM-based state updates for action-driven changes, and DovSG performs voxel-level local scene graph updates. However, all three handle only discrete changes — continuous or ambiguous state transitions and resolving contradictory observations across time remain open. DovSG's 35% absolute long-term task success rate highlights how far dynamic adaptation still needs to go for robust real-world deployment.
- **Scalability of LLM-dependent components**: Heavy reliance on LLMs for captioning, summarization, and retrieval creates cost and latency bottlenecks at scale. EmbodiedLGR shows that lightweight VLMs (<1B) can partially address this, but at significant accuracy cost (33.90% vs. R⁴'s 70.25% on comparable tasks).
- **Cross-environment generalization**: Whether spatial-semantic clustering hyperparameters (distance thresholds, similarity weights) transfer across radically different environment types.
- **Explore-vs-recall optimization**: Mind Palace introduces VoI stopping but the broader question of optimal policies for interleaving memory retrieval with physical exploration remains open, especially for questions requiring future prediction (past-present-future type).
- **Unifying navigation and manipulation memory**: Current architectures are task-specialized—non-parametric LLM pipelines for navigation/QA, differentiable parametric memory for manipulation. Whether a unified memory architecture can serve both remains a major open question.
- **Multi-round memory retrieval**: LMEE's MemoryExplorer is limited to single-round tool invocation; complex multi-step reasoning chains requiring iterative retrieval remain unsolved.
- **Belief revision under environmental change**: Theory of Space's false belief paradigm reveals that foundation models suffer from belief inertia — retaining obsolete spatial priors even after observing updated configurations. This is particularly relevant for embodied memory systems operating in dynamic environments (cf. DovSG, Embodied VideoAgent), where memory update mechanisms must actively overwrite stale information.
- **When does map-free break?**: Memory Over Maps underperforms HOV-SG on the Nfv scene (71.6% vs. 82.4%), suggesting reconstruction retains advantages in certain environments. Whether map-free approaches extend to manipulation (where Chameleon argues fine-grained perceptual memory is needed) or to environments with significant viewpoint changes between scanning and deployment is untested.

## Contradictions and debates

- **Schema-free vs. structured representations**: Embodied-RAG argues against fixed schemas (as in Hydra/3D scene graphs), but structured schemas may offer stronger guarantees for specific tasks. The trade-off between generality and task-specific performance is unresolved.
- **Hierarchical abstraction vs. flat retrieval**: Embodied-RAG builds explicit hierarchical structure (semantic forest) for multi-resolution reasoning, while ReMEmbR uses flat vector database retrieval with iterative LLM reasoning. R⁴ takes a middle path with structured per-object records queryable along three axes. Whether hierarchical memory construction, flat retrieval, or structured object databases is most effective likely depends on query type and environment scale.
- **Topological vs. metric spatial grounding**: Embodied-RAG uses topological graphs (abstract connectivity), while R⁴ uses SLAM-based metric coordinates (precise geometry). Metric grounding enables precise spatial reasoning but depends on SLAM reliability; topological approaches are more robust to localization errors but sacrifice geometric precision.
- **Continuous vs. episodic memory**: R⁴ maintains a single continuous database where all observations merge into one world model, while Mind Palace explicitly preserves temporal episode boundaries. Continuous memory enables object tracking but loses temporal context; episodic memory preserves "when" but may duplicate objects across episodes. The right approach likely depends on whether the application prioritizes spatial precision or temporal reasoning.
- **Structured extraction vs. raw image storage**: Most systems (Embodied-RAG, ReMEmbR, R⁴, Mind Palace) extract structured representations from observations, inevitably losing information. SnapMem keeps raw images and defers interpretation to VLMs, preserving spatial nuance but requiring more storage and computation at query time. This mirrors a broader debate in AI between eager vs. lazy feature extraction.
- **Non-parametric retrieval vs. learned recall**: The five navigation/QA systems all use non-parametric memory with similarity or LLM-based retrieval. Chameleon argues this paradigm fails under perceptual aliasing and proposes learned, goal-directed recall via differentiable memory trained with imagination objectives. This is the deepest architectural divide in the topic: whether memory access should be engineered (retrieval) or learned (recall).
- **Training-free generality vs. task-specific optimization**: R⁴ and Embodied-RAG achieve strong results without task-specific training, enabling zero-shot deployment. Chameleon requires end-to-end training per task but achieves superior disambiguation. LMEE's MemoryExplorer occupies a middle ground — training only the retrieval behavior via RL while keeping the foundation model's knowledge intact. The generality-specialization trade-off maps directly onto the non-parametric vs. parametric memory divide.
- **Passive vs. active memory**: Most systems (3D-Mem, SnapMem, R⁴) construct memory passively during exploration and query it post-hoc. LMEE argues agents must learn to actively invoke retrieval during exploration to guide future actions. Mind Palace's VoI stopping is a decision-theoretic approach to the same problem. MemoryEQA occupies a middle ground — engineered memory infusion into all modules without RL training. The optimal balance between passive accumulation and active querying remains unclear.
- **Learned vs. engineered retrieval policies**: LMEE trains retrieval behavior via RL (GRPO), while MemoryEQA achieves similar goals through entropy-based heuristics and module-specific queries. STaR uses Information Bottleneck. Whether learned policies outperform engineered ones at scale, and whether the approaches are complementary (e.g., RL-trained retrieval with entropy-based k-selection), remains untested.
- **Reconstruction vs. on-demand geometry**: Memory Over Maps argues that persistent 3D reconstruction is unnecessary for object localization — on-demand depth backprojection from stored keyframes achieves comparable accuracy at 100x+ lower preprocessing cost. HOV-SG invests hours of offline reconstruction but gains multi-level language grounding (floor→room→object queries) that map-free approaches cannot yet support; Memory Over Maps indexes scenes in ~1 minute but is limited to object-level queries. The trade-off depends on whether the application requires hierarchical spatial reasoning or just object localization.
- **Accuracy vs. efficiency**: Most systems use heavy models (GPT-4o, SAM2, VILA-13B) and achieve high accuracy but cannot run on-robot. EmbodiedLGR trades accuracy for real-time edge deployment with Florence-2 (<1B), achieving ~50% lower latency but significantly lower accuracy. STaR achieves a middle ground with ChatGPT-4.1-mini and ~30s stable query time. This reveals a Pareto frontier that the field has not yet mapped.
- **Redundancy as a first-class problem**: STaR explicitly argues that retrieval redundancy (not just retrieval accuracy) is a key bottleneck—redundant evidence increases hallucination risk. IB provides a principled solution; EmbodiedLGR uses threshold-based deduplication. Both suggest the field needs to move beyond "retrieve more" toward "retrieve better."
- **Map-free as a viable paradigm**: Memory Over Maps demonstrates that eliminating 3D reconstruction entirely — storing only posed RGB-D keyframes and performing on-demand depth backprojection at query time — can match or exceed reconstruction-based methods (HOV-SG, OpenFunGraph, VLMaps) on localization benchmarks while being 100x+ faster to build. This is the logical endpoint of the "images as memory" trend, suggesting that persistent 3D scene representations may be over-engineered for object-goal navigation.
- **Hybrid retrieval for scalable reasoning**: Memory Over Maps' two-stage SigLIP2+FAISS → VLM re-ranking pipeline demonstrates how to combine sub-millisecond embedding search with VLM reasoning over only K candidates, making foundation-model reasoning feasible at scale without exhaustive evaluation of all keyframes.
- **Memory-centric over planner-centric EQA**: MemoryEQA demonstrates that infusing retrieved memory into all modules (planner, stopping, answering) rather than just the answering stage yields 9.9% improvement on multi-target tasks. The entropy-based adaptive retrieval provides a principled mechanism for balancing retrieval precision and recall based on query complexity, complementing STaR's IB approach (which optimizes memory-side redundancy) and LMEE's RL-trained retrieval (which learns when to retrieve).
- **Viewpoint-contrastive memory updates**: MemoryEQA's update rule (position/rotation thresholds + structural/semantic similarity) addresses the memory corruption problem — noisy updates from redundant or similar observations degrading retrieval quality. This is a finer-grained version of EmbodiedLGR's spatial/semantic deduplication.
- **Scene graphs as LLM-grounding memory**: MoMa-LLM shows that dynamically built scene graphs serve as effective memory structures for LLM-based interactive search, with structured compact encoding enabling 97.7% success rate. The dynamic re-alignment of action history as room labels change addresses the memory consistency problem without explicit deduplication.
- **Inspection EQA as domain expansion**: BridgeEQA demonstrates that EQA extends far beyond household environments. With 48+ images per scene, hierarchical structural components, and NBI condition rubrics, bridge inspection stresses capabilities (multi-scale reasoning, evidence citation, expert alignment) that household benchmarks barely test. EMVR's MDP-based scene graph traversal converts EM-EQA to A-EQA, using navigational retrieval to mitigate "lost in the middle" bias — achieving +20.2pp Image Citation Relevance over feeding all images at once.
- **Image-based vs. object-based scene graphs**: BridgeEQA uses images as graph nodes (due to lack of bridge component detectors), contrasting with Hydra/HOV-SG/MoMa-LLM's object-centric nodes. This reveals that image-level nodes are a viable fallback when domain-specific perception is immature, though at the cost of coarser spatial reasoning.

## Recommended reading order

1. **ReMEmbR** (Anwar et al., 2024) — Accessible introduction to the problem; clearly motivates why long-horizon memory needs retrieval, introduces spatio-temporal query types.
2. **MoMa-LLM** (Honerkamp et al., 2024) — How to ground LLMs in dynamic scene graphs for interactive search; structured knowledge extraction shows that representation format matters as much as content. Same group as HOV-SG.
3. **HOV-SG** (Werby et al., 2024) — The strongest case for reconstruction-based memory: hierarchical open-vocabulary scene graphs enabling multi-level language grounding. Establishes the baseline that map-free approaches must beat.
4. **SnapMem** (Yang et al., 2024) — Contrasting approach: images as memory primitives with co-visibility clustering; good for understanding what structured representations lose.
5. **Embodied-RAG** (Xie et al., 2025) — Builds on the retrieval idea with hierarchical spatial abstraction; more comprehensive evaluation across environments and platforms.
6. **R⁴** (Sohn et al., 2025) — Most complete system for single-episode 4D reasoning: metric-anchored object database with SLAM integration, multi-agent sharing, and SOTA results approaching human performance.
7. **Mind Palace** (Ginting et al., 2025) — Extends to long-term multi-episode settings; introduces the explore-vs-recall trade-off and decision-theoretic stopping criteria. Most practically demonstrated system (real-world legged robot, 6 months of data).
8. **STaR** (Yuan et al., 2026) — Introduces principled non-redundant retrieval via Information Bottleneck; OmniMem's tripartite design shows what complementary memory modalities buy you.
9. **EmbodiedLGR** (Riva et al., 2026) — Practical counterpoint: what happens when you optimize for deployment efficiency and edge compute rather than accuracy. Exposes the implicit compute assumptions of all other systems.
10. **LMEE/MemoryExplorer** (Wang et al., 2026) — RL-trained active memory retrieval during multi-goal navigation; introduces the benchmark that unifies exploration process with outcome evaluation.
11. **Embodied VideoAgent** (Fan et al., 2025) — Bridges video understanding agents with embodied sensing; persistent object memory with VLM-based state updates for dynamic scenes. Good companion to R⁴ — both maintain per-object structured records but with different update philosophies (passive vs. action-driven).
12. **MemoryEQA** (Zhai et al., 2025) — Memory-centric EQA: infusing memory into all modules, not just answering. Entropy-based retrieval and viewpoint-contrastive updates provide engineered alternatives to LMEE's RL-trained approach. Read alongside LMEE for the learned-vs-engineered debate.
13. **Memory Over Maps** (Zhou et al., 2026) — The most extreme anti-reconstruction position: no 3D map at all, just posed keyframes with on-demand geometry. Shows when and why map-free works — and where it doesn't. Also introduces a clean hybrid retrieval design (embedding + VLM re-ranking).
14. **DovSG** (Yan et al., 2025) — Dynamic scene graph updates for real-hardware mobile manipulation; local subgraph update mechanism shows how memory update granularity impacts long-term task success. The only system in this topic validated with end-to-end manipulation on physical robots.
15. **Chameleon** (Guo et al., 2026) — Read last as a counterpoint: fully differentiable bio-inspired memory for manipulation that critiques the entire non-parametric/LLM-pipeline paradigm. Essential for understanding the limits of semantic compression and similarity-based retrieval.
