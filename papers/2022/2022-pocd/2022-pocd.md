# POCD: Probabilistic Object-Level Change Detection and Volumetric Mapping in Semi-Static Scenes

![[2022-pocd-thumbnail.png]]

## Research gap

POCD introduces a probabilistic object-level change detection framework for maintaining volumetric maps (TSDFs) in semi-static environments — scenes that are primarily static but where objects shift, appear, or disappear between robot traversals. Traditional TSDF methods assume i.i.d. voxel occupancy and corrupt when the scene changes, leaving ghosting artifacts. POCD addresses this by jointly modeling a stationarity score and a TSDF change magnitude for each detected object, propagating detected changes across entire objects consistently.

## Contributions

- Bayesian object-level state representation jointly modeling stationarity probability and geometric change magnitude via TSDF voxel differences
- Update rule leveraging both geometric (TSDF difference) and semantic (object class) measurements for robust change detection in incrementally evolving scenes
- Online, object-aware mapping framework that propagates detected changes across entire objects for consistent map maintenance
- TorWIC dataset: real-world warehouse change detection benchmark with RGB-D, LiDAR, and robot poses across multiple traversals with object rearrangements
- First system for online, probabilistic, object-level change detection in large indoor semi-static environments

## Method

**Object Representation**: Each tracked object has a per-object TSDF, a stationarity score s ∈ [0,1], and a geometric change magnitude g. The joint probability p(s,g) is updated via Bayesian inference as new observations arrive.

**Geometric Change Measure**: Sum of voxel-wise TSDF value differences between the object's stored TSDF and the newly observed TSDF within the object's bounding volume. Large differences indicate the object has moved or been removed.

**Semantic Consistency**: Object class labels from 2D semantic segmentation are projected into 3D. If the detected class within an object's volume changes, this provides additional evidence of change.

**Bayesian Update**: p(s_t | z_{1:t}) ∝ p(z_t | s_t) · p(s_t | s_{t-1}) — measurement likelihood combines geometric and semantic evidence; transition prior models expected stationarity persistence. When stationarity drops below threshold, the object's TSDF voxels are cleared.

**Map Maintenance**: Cleared voxels are re-integrated from new observations, producing a clean map reflecting the current state without ghosting artifacts from previous object positions.

## Datasets & evaluation

Evaluated on TorWIC (warehouse, 5 routes with incremental object changes) and ToyCar (tabletop). POCD produces significantly cleaner maps than traditional TSDF (voxblox) and Kimera, with no ghosting artifacts from moved objects. Outperforms MaskFusion and TSDF++ on reconstruction quality in semi-static scenes. Robust to sensor noise, partial observations, and dynamic objects (people walking through). The Bayesian update correctly distinguishes between objects that have truly changed vs. those with noisy measurements.

## Limitations

- Requires object detection/segmentation to identify objects for tracking — detection quality directly limits change detection quality
- TSDF voxel difference as change measure may be sensitive to registration errors and noise, especially at object boundaries
- Stationarity threshold requires tuning; too aggressive → false positives (clearing unchanged objects), too conservative → missed changes
- No semantic understanding of what changed — knows an object moved but not what it is or where it went (unlike 3DGS-CD's 6D pose estimation)
- Limited to rigid objects — deformable or articulated objects not handled
- TorWIC dataset, while valuable, is limited to a single warehouse environment

## Key takeaways

The core innovation is a **Bayesian object-level update rule** that combines geometric measurements (sum of voxel differences between before/after TSDFs within an object's bounding volume) with semantic measurements (object class consistency) to maintain a per-object stationarity probability. When an object's stationarity drops below a threshold, its TSDF voxels are cleared, allowing new observations to reconstruct the updated scene without artifacts.

The method operates online, integrating new observations incrementally as the robot revisits the environment. It handles objects that shift, move significantly, appear, or disappear — all without requiring continuous observation of the transition (unlike dynamic SLAM methods that track frame-to-frame motion).

The paper also introduces the Toronto Warehouse Incremental Change Dataset (TorWIC), a real-world change detection dataset captured in a warehouse with RGB-D camera and 2D LiDAR, filling a gap in large-scale industrial semi-static evaluation.

POCD addresses the semi-static gap between static SLAM (Kimera, voxblox — assume world doesn't change) and dynamic SLAM (MaskFusion, MID-Fusion — track frame-to-frame motion). Semi-static changes are unobserved transitions that occur while the robot is offline — a fundamentally different challenge from continuous dynamics.

Within the change detection topic, POCD represents the **volumetric/TSDF approach** to 3D change detection, complementing the radiance-field-based approaches (3DGS-CD, Online SCD). Where 3DGS-CD uses 3DGS rendering for pre/post comparison and Online SCD uses self-supervised fusion, POCD directly compares TSDF voxel values — a simpler but effective geometric approach. The Bayesian object-level reasoning anticipates SceneDiff's (2026) instance-level change classification but operates in voxel space rather than feature space.

The warehouse setting connects to practical robotics applications — long-term deployment where the environment changes slowly. This is the same motivation as embodied memory systems (R⁴, ReMEmbR) but focused on maintaining geometric consistency rather than semantic queryability.
