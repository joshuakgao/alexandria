# Mask3D: Mask Transformer for 3D Semantic Instance Segmentation

![[2023-mask3d-thumbnail.png]]

## Research gap

Mask3D is the first competitive Transformer-based model for 3D semantic instance segmentation on point clouds. It replaces the hand-crafted voting and clustering pipelines that dominated the field with a clean, end-to-end architecture inspired by Mask2Former. Each object instance is represented as a learned instance query that iteratively attends to multi-scale point cloud features via a Transformer decoder, directly producing instance masks in parallel without center voting, non-maximum suppression, or geometric grouping heuristics.

## Contributions

- First competitive Transformer-based approach for 3D semantic instance segmentation, eliminating voting schemes, geometric grouping, and hand-tuned hyperparameters
- Instance queries that encode both semantic and geometric information, directly predicting masks via dot product with point features
- Masked cross-attention that restricts each query to attend only within its predicted mask region, improving instance-level focus
- Sampled cross-attention that randomly subsamples voxel features per decoder layer during training, enabling higher-resolution voxel grids within memory constraints
- Non-parametric queries (initialized from farthest point sampling) enabling variable query counts at inference without retraining
- State-of-the-art on ScanNet, ScanNet200, S3DIS, and STPLS3D

## Method

**Feature Backbone**: MinkowskiEngine Res16UNet34C processes the colored point cloud (voxelized at 2cm) into multi-scale feature maps F_r at resolutions r=0,...,R. Full-resolution F_0 is used for mask prediction; coarser levels for cross-attention.

**Instance Queries**: K=150 queries (non-parametric: initialized from farthest point sampling positions with zero features + Fourier positional encodings). Refined through L=12 Transformer decoder layers attending to 4 feature scales × 3 iterations, with shared weights across iterations.

**Mask Module**: Maps instance queries through MLP to match point feature space, then computes dot product with F_0 → sigmoid → threshold at 0.5 for binary masks. Also predicts semantic class via linear projection to C+1 dimensions.

**Masked Cross-Attention**: Each instance query attends only to voxels within its intermediate predicted mask (−∞ masking for out-of-mask voxels), forcing instance-level focus.

**Training**: Bipartite Hungarian matching for supervision assignment. Combined Dice + BCE mask loss + cross-entropy classification loss. Auxiliary losses at each decoder layer. Trained for 600 epochs on NVIDIA A40 (~78 hours).

**Confidence**: Product of softmax class confidence and mean mask confidence of in-mask voxels.

## Datasets & evaluation

ScanNet test: 56.6 mAP / 78.0 mAP50 (previous best: 50.4/76.1 SoftGroup). ScanNet val: 55.2 mAP / 73.7 mAP50. S3DIS 6-fold: +10.1 mAP over prior art. STPLS3D: +11.2 mAP. ScanNet200 test: +12.4 mAP. Runtime: 339ms per scene on TITAN X, comparable to SoftGroup (345ms) and HAIS (339ms). Non-parametric queries enable trading inference speed for performance by adjusting query count at test time.

## Limitations

- Closed-vocabulary: trained on fixed category sets (18 for ScanNet, 13 for S3DIS); does not support open-vocabulary queries — later work (OpenMask3D, Dual-Path) addresses this by adding CLIP features
- Performance drops significantly on long-tail categories (ScanNet200 shows this clearly), though still SOTA
- Voxelization at 2cm resolution limits fine-grained detail; finer voxels require more memory
- Sampled cross-attention during training is a form of dropout that may miss small instances
- The fixed query count K=150 must be set to exceed the maximum expected instance count; insufficient queries miss objects

## Key takeaways

The model uses a sparse convolutional U-Net backbone (MinkowskiEngine) to extract multi-scale point features, then a stack of L=12 Transformer decoder layers refines K instance queries through cross-attention to point features (coarse-to-fine across 4 resolution levels, repeated 3 times) and self-attention among queries. A mask module computes dot products between instance query features and full-resolution point features to produce instance heatmaps, which are thresholded into binary masks. Foreground-background masked cross-attention (from Mask2Former) focuses each query on its predicted instance region, and bipartite Hungarian matching provides optimal supervision assignment during training.

Mask3D sets new state-of-the-art on four benchmarks: ScanNet test (+6.2 mAP over prior SOTA), S3DIS 6-fold (+10.1 mAP), STPLS3D (+11.2 mAP), and ScanNet200 test (+12.4 mAP), while running at comparable speed to prior methods (339ms per scene).

Mask3D brings the Mask2Former paradigm from 2D to 3D instance segmentation, replacing the dominant center-voting + clustering pipeline (VoteNet → PointGroup → SoftGroup → HAIS) with a cleaner Transformer-based approach. Where prior methods require hand-tuned geometric properties (center offsets, grouping radii, score networks), Mask3D uses domain-agnostic Transformer building blocks with bipartite matching.

Within the 3D scene understanding topic, Mask3D is a foundational 3D-native method that complements the 2D-to-3D lifting approaches (SAM3D, Dual-Path Framework). SAM3D and the Dual-Path Framework use Mask3D or similar 3D models as their point cloud pathway, making Mask3D's quality directly impact downstream zero-shot and open-vocabulary systems.
