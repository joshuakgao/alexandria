# VGG-T³: Offline Feed-Forward 3D Reconstruction at Scale

![[2026-vgg-t3-thumbnail.png]]

## Research gap

VGG-T³ (Visual Geometry Grounded Test Time Training) solves a fundamental scalability bottleneck in offline feed-forward 3D reconstruction: the quadratic complexity of global self-attention w.r.t. the number of input views. The key insight is that the variable-length Key-Value (KV) space in VGGT's global attention layers functions as an implicit scene representation, and this can be **compressed into a fixed-size MLP via test-time training (TTT)**. This converts the quadratic O(n²) attention operation into a linear O(n) MLP application, while retaining the global scene aggregation capability of offline methods.

## Contributions

- Distilling VGGT's variable-length KV scene representation into a fixed-size MLP via test-time training, achieving linear O(n) scaling
- Post-training linearization of a pre-trained quadratic model at ~12% of original training cost
- Mini-batch TTT enabling single-GPU inference on arbitrarily large image collections and linear distributed scaling
- Unified mapping and localization: the optimized MLP serves as a queryable scene representation for feed-forward visual localization
- ShortConv2D spatial mixing to break trivial K→V linear solutions and improve reconstruction quality

## Method

Built on VGGT's architecture of alternating image-wise self-attention and global self-attention blocks:

1. **Linearization**: Replace all global attention layers with TTT layers. Initialize from pre-trained VGGT weights (Wq, Wk, Wv). Replace LayerNorm with L2 normalization.
2. **TTT objective**: Learn MLP weights θ by minimizing dot-product loss between Tθ(ki) and vi across all input tokens. Uses SwiGLU MLP architecture and Muon optimizer.
3. **ShortConv2D**: Apply 3×3 spatial convolution on values V before TTT optimization, creating non-linear V' that prevents trivial solutions from the linear K→V relationship.
4. **Mini-batch inference**: Total gradient is a sum of per-token gradients, enabling arbitrary sharding across mini-batches (for CPU offloading) or GPUs (for distributed inference).
5. **Visual localization**: After reconstruction, freeze MLP weights and run forward pass on novel query image — global attention layers apply frozen MLP to query features without updating θ.
6. **Entropy-scaling**: For fair comparison, apply logN-based scaling to softmax attention in VGGT baseline to improve its long-sequence generalization.

Fine-tuned on dataset comparable to VGGT's training data, 100k steps on 8× A100-80GB.

## Limitations

- Reconstruction quality degrades on scenes with larger spatial extent and complex layouts (Waymo failure cases), suggesting linear attention cannot yet match softmax in all settings
- Visual localization accuracy (30.64% at 20cm/20° on Wayspots) is still far below classical methods — proof-of-concept rather than competitive
- Only 2 TTT optimizer steps needed for training-size collections, but optimal step count for very large collections not fully characterized
- The fixed-size MLP has a capacity ceiling — whether it can represent arbitrarily complex scenes is untested
- Entropy-scaling of softmax attention in baseline VGGT substantially improves its large-collection performance, making the baseline comparison fairer but the improvement margin smaller

## Key takeaways

The approach works in two stages per global attention layer: (1) **Update** — project input tokens to queries, keys, and values, then optimize the MLP weights θ to learn the K→V mapping using a self-supervised TTT objective; (2) **Apply** — query the scene representation by applying the frozen MLP to query tokens. The method starts from pre-trained VGGT weights and replaces global attention with TTT layers, fine-tuning only these layers at ~12% of VGGT's original training cost. A critical technical contribution is replacing LayerNorm with L2 normalization for fast convergence from pre-trained weights, plus ShortConv2D on values to break the trivial linear K→V solution.

VGG-T³ reconstructs a 1k image collection in 54 seconds (11.6× faster than VGGT) and a 2k collection in 48.5 seconds (33× faster). It supports single-GPU inference for arbitrarily large collections via mini-batch gradient accumulation with CPU offloading, and distributed inference via gradient sharding. The compressed MLP scene representation also unlocks **feed-forward visual localization**: querying the frozen MLP with a novel view produces its pose relative to the reconstruction, unifying mapping and localization in one model.

On standard benchmarks, VGG-T³ outperforms the other linear-time method (TTT3R) by 2–2.5× on DTU, ETH3D, and NRGBD while remaining competitive with quadratic-time baselines (VGGT, SparseVGGT, FastVGGT). Visual localization on Wayspots achieves 30.64% at 20cm/20° vs. TTT3R's 2.94%.

Directly extends **VGGT** by replacing its quadratic global attention with linear TTT layers, maintaining the same encoder/decoder and pre-trained weights. Addresses the scalability limitations highlighted by **DAGE** (which uses dual-stream architecture) and **π³** (which shares the quadratic attention bottleneck) — VGG-T³ offers an orthogonal solution via representation compression rather than architectural redesign. Outperforms concurrent **TTT3R** (which builds on CUT3R with auto-regressive processing) by large margins on pointmap and localization tasks, demonstrating that offline global optimization is fundamentally more accurate than online sequential processing. The unified mapping + localization capability relates to **ACEZero** but replaces its extensive iterative optimization with a few TTT steps from pre-trained weights. **FastVGGT** (token merging) and **SparseVGGT** (block-sparse attention) are competing approaches to the same scalability problem but remain asymptotically quadratic.
