# From Image Hashing to Scene Change Detection

![[2026-hashscd-thumbnail.png]]

## Research gap

HashSCD is the first hashing-based framework for scene change detection, bridging image hashing and spatial change localization within a unified architecture. Rather than storing high-dimensional continuous features for each observed scene, HashSCD encodes spatially-aligned image patches into compact binary hash codes and aggregates them through a cascaded XOR operation to produce a single global hash per image. This enables both efficient global change detection (via Hamming distance on global hashes) and localized change identification (via patch-wise hash comparison and upsampling) — all without re-processing previously stored images through the backbone network.

## Contributions

- First hashing-based framework for scene change detection, supporting both global retrieval and localized change identification
- Patch-wise hash embedding with recursive XOR-like aggregation that preserves spatial information in a fixed-length binary code
- Unsupervised contrastive training at both patch and global levels, requiring no labeled change masks
- 256x reduction in storage and computation compared to continuous-feature methods, enabling scalable long-term monitoring

## Method

HashSCD uses a VGG16 backbone to extract a feature map F ∈ R^{C×H×W}, which is decomposed into P = H×W local feature vectors. Each patch feature is projected through a learnable hashing layer and mapped to a soft hash code via ϕ(·) = (tanh(·) + 1) / 2 during training. Patch-wise hashes are aggregated through a recursive absolute-difference operation (equivalent to cascaded bitwise XOR in binary domain) to produce a global hash R ∈ R^l.

Training uses asymmetric contrastive loss applied at both patch level (encouraging local spatial consistency across augmented views) and global level (on the aggregated hash). At inference, soft codes are binarized via sign function and all comparisons operate in Hamming space. Change heatmaps are generated by upsampling patch-level Hamming distances to input resolution.

For long-term monitoring, only (P+1)×l bits are stored per scene state — patch-wise and global hashes. New observations are compared against stored hashes without re-inference on historical images.

## Datasets & evaluation

On VL-CMU-CD: F1 = 0.694 (64-bit), compared to SSCD 0.745, D-SSCD 0.725, supervised C-3PO 0.794. On PCD: F1 = 0.695 (64-bit), compared to SSCD 0.583, D-SSCD 0.642. HashSCD outperforms both unsupervised baselines on PCD while maintaining competitive performance on VL-CMU-CD. Retrieval performance (mAP) reaches 0.660 on VL-CMU-CD and 0.933 on PCD at 64-bit, demonstrating dual retrieval+detection capability. Also achieves competitive retrieval results on standard hashing benchmarks (Oxford Flowers, Food101, NUS-WIDE).

## Limitations

- Performance gap vs. continuous-feature methods on noisy datasets (VL-CMU-CD) suggests binary codes may lose fine-grained appearance information needed to distinguish true changes from noise
- Only evaluated with VGG16 backbone; whether modern architectures (ViT, foundation models) improve hash quality is unexplored
- No evaluation on 3D change detection or multiview settings — currently limited to paired 2D images
- Hash length vs. accuracy trade-off: 64-bit outperforms 16-bit significantly, but diminishing returns suggest an optimal hash length may exist
- Viewpoint invariance not explicitly addressed; large viewpoint changes between T0 and T1 may degrade patch correspondence

## Key takeaways

The model is trained in a fully unsupervised manner using asymmetric contrastive learning at both patch and global levels, with a soft hashing strategy (tanh relaxation) enabling gradient-based optimization during training while producing hard binary codes at inference. Experiments on VL-CMU-CD and PCD benchmarks show competitive F1 scores against unsupervised continuous-feature methods (SSCD, D-SSCD) while achieving a 256-fold reduction in both storage (8 KB vs. 2 MB per image at 64-bit) and computational cost for distance computation.

HashSCD opens a fundamentally new direction for scene change detection — compact binary representations — that is orthogonal to all prior approaches in this wiki. Continuous-feature methods like SSCD/D-SSCD achieve higher detection accuracy but at 256x the storage and compute cost. Zero-shot methods (ZSSCD, GeSCF) use foundation models but are even more expensive. The hashing approach is most relevant to long-term, large-scale monitoring scenarios where O-SCD and Chamelion operate, but trades some detection accuracy for dramatic efficiency gains. The patch-wise formulation shares conceptual similarity with SceneDiff's spatial decomposition but operates in binary rather than continuous space.
