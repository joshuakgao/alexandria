# STaR: Scalable Task-Conditioned Retrieval for Long-Horizon Multimodal Robot Memory

![[2026-star-thumbnail.png]]

## Research gap

STaR addresses a critical weakness in existing embodied memory systems: naive retrieval returns redundant, task-irrelevant evidence that dilutes useful context and increases hallucination risk. The key innovation is applying the **Information Bottleneck (IB)** principle to produce compact, non-redundant, task-relevant memory subsets from a multimodal long-term memory.

## Contributions

- Introduces OmniMem: a tripartite multimodal memory (video captions + 3D primitives + visual keyframes) that preserves complementary information at different granularities
- Applies the Information Bottleneck principle for task-conditioned retrieval, producing compact evidence sets that suppress redundancy while preserving task-relevant information
- Proposes caption-induced primitive subsetting: only primitives linked to query-relevant captions are clustered, avoiding re-aggregation of the entire scene
- Achieves 2–4× lower spatial error than baselines on long-horizon NaVQA sequences with stable ~30s query time regardless of memory length
- Creates WH-VQA: a warehouse benchmark in Isaac Sim with visually similar objects requiring task-level disambiguation and multimodal queries
- Deploys on a real Husky robot in indoor and outdoor environments

## Method

**OmniMem Construction**: Video captions are generated by NVILA at M-second intervals, embedded and stored in Milvus. 3D primitives are built via OpenGraph (RAM + Grounding-DINO + TAP for perception, LiDAR filtered by 4DMOS, DBSCAN denoising) storing geometry, captions, semantic features, and timestamps. Keyframes are logged at ~1 Hz with overlaid object contours labeled by primitive index.

**STaR Retrieval**: (1) Query Q retrieves high-recall caption pool C_Q via vector similarity > threshold τ. (2) Captions induce a primitive working set X'_Q via their detection indices. (3) IB clusters X'_Q by iteratively merging adjacent primitives that minimize task-driven dissimilarity d_ij (Jensen-Shannon divergence weighted by cluster priors), stopping when fractional information loss δ(k) exceeds δ̄. Task relevance p(y|x') is computed via cosine similarity between primitive features and query embeddings. (4) Captions grouped by cluster; one representative per cluster selected by query relevance; top-K representatives form R_text. (5) LLM optionally selects keyframe timestamps from R_text for visual evidence R_img.

**Agentic Workflow**: MLLM planner (ChatGPT-4.1-mini) performs up to 3 reasoning rounds (~5.9s + 6.8s + 2.7s), issuing text/time/position/distance API calls. Answer generation (14.2s) uses retrieved multimodal evidence. Milvus retrieval is fast (~0.5s text, ≤0.1s other).

## Datasets & evaluation

- **NaVQA spatial (long)**: 0.77 success rate, 16.9m error (vs. ReMEmbR 0.49/53.9m, OpenGraph 0.26/87.4m)
- **NaVQA textual (long)**: 0.80 success rate (vs. ReMEmbR 0.50, OpenGraph 0.36)
- **Scalability**: Stable ~30s query time from 36s to 35.9min memory length
- **WH-VQA**: Superior performance on visually similar objects requiring disambiguation
- **Spatial grounding**: Answers grounded in 3D bounding-box centers rather than robot pose, yielding more actionable navigation goals
- **Non-redundancy**: IB clustering prevents retrieval collapse where top-K results all come from one local region

## Limitations

- IB clustering threshold δ̄ and top-k task scores require tuning; sensitivity analysis across diverse environments is limited
- Relies on OpenGraph's 3D reconstruction quality; incomplete or noisy primitives degrade clustering
- Visual keyframe selection is LLM-driven and may miss critical frames; no learned keyframe importance scoring
- WH-VQA is simulated (Isaac Sim); real warehouse evaluation with truly visually similar objects at scale is needed
- The agentic workflow uses up to 3 reasoning rounds, adding latency (~30s total); not yet suitable for real-time HRI at the speed EmbodiedLGR targets
- Memory construction assumes access to depth/LiDAR; applicability to RGB-only setups (common in consumer robotics) is unclear

## Key takeaways

The system builds **OmniMem**, a tripartite memory comprising: (1) **video captions** C (NVILA-generated scene descriptions stored in Milvus vector DB with timestamps and poses), (2) **3D geometric primitives** X (open-vocabulary 3D segments from OpenGraph with point clouds, captions, semantic features, and detection timestamps), and (3) **visual keyframes** K (annotated images with object contours and primitive indices). This three-level design captures complementary information: captions for mid-level dynamics, primitives for spatial structure, and keyframes for fine-grained visual detail.

At query time, **STaR retrieval** operates in three stages: (1) video captions semantically matching the query induce a working set of 3D primitives; (2) IB iteratively merges neighboring primitives into task-relevant clusters, stopping when information loss exceeds a threshold; (3) captions are grouped by cluster, and one representative caption per cluster is selected, producing a diverse, non-redundant evidence set. When finer detail is needed, the LLM selects keyframe timestamps from the textual evidence to retrieve images.

An **agentic RAG workflow** with an MLLM planner performs up to three reasoning rounds, issuing API calls (text, time, position, distance queries) against OmniMem. Evaluated on NaVQA and WH-VQA (a warehouse benchmark with many visually similar objects), STaR consistently outperforms ReMEmbR and OpenGraph baselines, achieving 2–4× lower spatial error on long-horizon sequences. Real-world deployment on a Husky robot demonstrates practical utility.

STaR directly builds on [[2024-remembr|ReMEmbR]], using the same NaVQA benchmark and similar query types, but replaces ReMEmbR's flat vector DB retrieval with IB-based task-conditioned clustering over 3D primitives. Where ReMEmbR retrieves captions by embedding similarity (producing redundant results from revisited locations), STaR ensures diversity by selecting one representative per task-relevant spatial cluster. The OmniMem design echoes [[2025-r4-retrieval-augmented-reasoning-4d|R⁴]]'s separation into semantic/spatial/temporal backends but adds visual keyframes as a third modality and uses IB rather than SLAM-anchored object matching for deduplication. Compared to [[2026-embodied-lgr|EmbodiedLGR]]'s graph-based deduplication, STaR's IB approach is more principled—it optimizes an information-theoretic objective rather than applying fixed thresholds. The caption-induced primitive subsetting avoids [[2025-embodied-rag|Embodied-RAG]]'s need to cluster the entire scene, enabling scalability with growing memory.
