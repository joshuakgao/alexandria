# Explore with Long-term Memory: A Benchmark and Multimodal LLM-based Reinforcement Learning Framework for Embodied Exploration

![[2026-lmee-long-term-memory-embodied-exploration-thumbnail.png]]

## Research gap

LMEE (Long-term Memory Embodied Exploration) addresses a key gap in embodied AI: existing benchmarks focus on task completion outcomes but neglect the exploration process itself and how agents build and use memory during navigation. The paper argues that an ideal embodied agent should not only complete goals but also accumulate episodic memories that support future reasoning — a prerequisite for lifelong learning.

## Contributions

- LMEE task formulation that unifies multi-goal navigation with memory-based QA, evaluating both exploration process and outcome
- LMEE-Bench: large-scale benchmark with 1,982 tasks, 9,286 questions across 5 types (attribute, counting, location, relationship, state), and 3 difficulty levels
- MemoryExplorer: MLLM agent trained via GRPO reinforcement learning to proactively invoke memory retrieval tools during exploration
- Multi-task reward function integrating action accuracy, frontier correctness, answer precision, and format compliance with tool-use incentivization
- Significant improvements over state-of-the-art exploration models (Explore-EQA, 3D-Mem) on both navigation success rate and QA accuracy

## Method

**LMEE-Bench Data Construction:**
1. Task instructions generated by Qwen3-235B from HM3D scene objects and regions
2. Exploration trajectories generated via Habitat-Sim path planning with step-by-step actions, observations, and positions
3. Memory bank constructed with CLIP-encoded text/image features and positions, using adaptive similarity thresholding for retrieval
4. QA pairs generated by Qwen3-VL-235B from observation images of navigation targets

**MemoryExplorer:**
- **Input**: Task instruction, multi-view observations (left/front/right), goal-oriented question, external memory bank
- **Memory Retrieval**: Model generates code to invoke retrieval tool with query text; CLIP-based cosine similarity retrieves top-k memories (text + image)
- **Output**: Single-step action, frontier selection, question answer
- **Training**: GRPO reinforcement fine-tuning on Qwen2.5-VL-7B with multi-task reward:
  - `r_total = w_act · r_action · c + w_front · r_frontier · c + w_ans · r_answer + w_fmt · r_format`
  - Scaling factor α amplifies rewards when tool is successfully invoked, reduces them on failure — incentivizing active tool use
- **Consistency coefficient** c penalizes logically inconsistent action-frontier pairs

## Datasets & evaluation

- **LMEE-Bench**: MemoryExplorer achieves 22.22% SR (vs. 3D-Mem 14.81%, Explore-EQA 7.41%) and 47.59% QA accuracy (vs. 3D-Mem 39.66%)
- Strongest gains on hard tasks where long-horizon memory retrieval is critical
- On **GOAT-Bench**: Competitive with 3D-Mem on multimodal lifelong navigation
- Retrieval-augmented QA baselines (GPT-4o, Gemini-2.5-Flash, Qwen3-VL) achieve reasonable QA scores but cannot navigate — MemoryExplorer unifies both capabilities
- Ablation shows all reward components contribute; removing tool-use incentivization significantly hurts QA accuracy

## Limitations

- Training and evaluation on HM3D scenes only — generalization to real-world environments untested
- Single-round tool invocation limits complex multi-step reasoning chains
- Memory retrieval depends on CLIP similarity, which may miss semantically relevant but visually dissimilar memories
- The continuous action window sampling may discard important fine-grained movement decisions
- QA evaluation limited to navigation-target-related questions — broader scene understanding not assessed
- Scale: 7B model may struggle with very long-horizon tasks requiring more complex reasoning

## Key takeaways

The contribution is twofold. First, the authors propose the LMEE task formulation and LMEE-Bench, a benchmark that unifies multi-goal navigation with memory-based question answering. An agent navigates to multiple targets in sequence (e.g., "check the Christmas tree, then the dryer, then the nightstand"), dynamically building an episodic memory bank along the way. After navigation, the agent must answer questions about observed scenes (e.g., "Is the washing machine door open or closed?") by retrieving from its accumulated memories. The benchmark includes 246 object categories, 9,000+ goals and questions, and 1,982 exploration trajectories across HM3D scenes, with easy/medium/hard difficulty levels.

Second, they propose MemoryExplorer, an MLLM-based agent (built on Qwen2.5-VL-7B) trained via reinforcement learning (GRPO) to actively query and retrieve memories during exploration. The key innovation is that the model learns to proactively invoke a memory retrieval tool — generating code to call `retrieve_memories(memory, query_text=...)` — and use retrieved multimodal memories to reason about both task progress and questions. A multi-task reward function jointly optimizes action prediction, frontier selection, and question answering, encouraging the agent to explore while simultaneously building useful memories.

LMEE directly addresses limitations of several papers in this wiki:

**3D-Mem** and **SnapMem** construct memory snapshots for reasoning but use passive memory — they don't learn to proactively retrieve. MemoryExplorer's RL-based active tool invocation is a direct response.

**Mind Palace** performs post-exploration reasoning with episodic world instances but doesn't integrate memory retrieval into the exploration loop itself. LMEE's multi-task reward unifies exploration and memory use.

**Embodied-RAG** provides general-purpose retrieval over semantic forests but focuses on single-goal tasks. LMEE extends to multi-goal long-horizon settings.

**R⁴** builds 4D knowledge databases for training-free retrieval but does not learn proactive querying. MemoryExplorer shows that RL fine-tuning specifically to encourage tool invocation yields better memory utilization.

The LMEE benchmark is the first to combine multi-goal navigation with memory-based QA, filling a gap left by existing benchmarks (ObjectNav, GOAT-Bench, OpenEQA) that evaluate either navigation or QA but not both simultaneously.
